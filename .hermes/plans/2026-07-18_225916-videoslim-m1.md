# VideoSlim M1 End-to-End Prototype Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Deliver an ARM64 APK that imports one local video, reads its technical metadata, compresses it with a fixed balanced HEVC preset, saves the result to `Movies/VideoSlim`, and exposes complete in-app debug logs.

**Architecture:** Keep Flutter UI/business code isolated from a strict `VideoEngine` interface. Implement Android behavior in Kotlin behind `videoslim/engine` and `videoslim/progress`; keep F19 persistence and native sharing behind `videoslim/logs`. Media3 writes to an app-private temporary MP4 and only publishes a completed file through MediaStore.

**Tech Stack:** Flutter 3.44.6 / Dart 3.12, Provider, Kotlin 2.3.20, Android API 26–36, AndroidX Activity 1.13.0, Jetpack Media3 1.10.1 (`common`, `effect`, `transformer`), JUnit 4.

---

## M1 scope gate

In scope only:

- F1: Photo Picker video import plus SAF file import, content URI end-to-end.
- F2: metadata page from `MediaMetadataRetriever` + `MediaExtractor`, including display-oriented dimensions and HDR detection.
- F3 M1: fixed balanced preset only: HEVC, 2.5 Mbps, original resolution, audio copy.
- Simple in-app progress dialog via EventChannel; no foreground service and no cancellation UI.
- F7 M1: publish successful output to `Movies/VideoSlim` and show a basic result.
- F19: Flutter/native/platform-call/flow logging, private-file persistence, 2000-entry/1 MB rotation, copy/share.

Explicitly out of scope: configurable presets, automatic H.264 fallback, crop, trim, audio extraction, background service, notification, wake lock, cancellation UI, history, batch queue, target size, and full result-page actions.

## Technical decisions

1. State management is **Provider** and remains the selected project convention.
2. Media3 dependencies use stable `1.10.1`; prerelease `1.11.0-beta01` is excluded.
3. No `INTERNET` permission is declared.
4. API 29+ MediaStore uses `RELATIVE_PATH` + `IS_PENDING` without storage permission.
5. Android 8–9 cannot publish automatically to public `Movies/` without legacy write access. To preserve minSdk 26 and F7 behavior, declare/request `WRITE_EXTERNAL_STORAGE` with `maxSdkVersion=28` only. Modern Android never receives this permission request. This is an official-platform compatibility correction to the PRD's blanket no-storage-permission sentence, not a change to F1.
6. Version advances to `1.1.0+2`; the same debug certificate enables update-in-place over M0.
7. M1 rejects HDR inputs before processing with an explicit user message, per R4.

## Residual physical-device gate

Only the owner can prove on real hardware:

- Photo Picker and SAF import both work.
- Portrait dimensions and displayed metadata match the selected video.
- Typical SDR phone video compresses, output appears immediately under `Movies/VideoSlim`, plays in gallery, and is materially smaller.
- A corrupt/unsupported input produces a complete readable F19 chain.
- Log copy and Android share chooser both work.

M2 must not start until this gate passes.
