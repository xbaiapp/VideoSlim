## 2. 素材准备

至少准备：

- A：10–30秒、画面和音频都有明显时间标记的视频；
- B：2分钟以上、开头和结尾内容明显不同的视频；
- C：rotation 90°或270°的竖拍视频；
- D：有可靠拍摄时间和位置的来源；
- E：无拍摄时间和位置的来源；
- F：足够长、可测试后台、锁屏、取消和进程恢复的视频；
- G：AAC音轨来源，以及可验证AAC重编码和移除音轨的来源。

对每个素材记录文件名、时长、帧率、显示方向、video/audio codec、源大小和可识别的预期起止画面。可用`ffprobe`记录源和输出的首尾PTS，但不得只用metadata替代主观播放检查。

## 3. 每个成功输出的共同检查

- [ ] 输出只包含所选半开区间`[startMs,endMs)`对应的连续内容；
- [ ] 从头、中间、结尾正常播放，画面连续且音画同步；
- [ ] 页面和通知显示的终态、文件名、保存位置一致；
- [ ] 输出可以打开和分享；
- [ ] 源视频仍存在、可播放且字节未被修改；
- [ ] 旧输出和VideoSlim以外的文件没有被覆盖、修改或删除；
- [ ] 默认MediaStore与自定义SAF都不留下失败/取消半成品；
- [ ] 完成后可以开始下一项任务。

## 7. 停止条件

出现任一情况立即停止并保留素材、步骤、截图/录屏、输出和已脱敏F19日志：

1. 输出包含区间外内容、缺少区间内明显内容或出现非连续拼接；
2. 可感知音画漂移、首尾卡顿、闪帧、黑帧或无法播放；
3. retry/recovery丢失或改变端点，或产生重复任务/重复输出；
4. crop、缩放、codec、码率或音频设置被trim静默改变；
5. 源文件、旧输出或其他文件丢失、覆盖、修改或误删；
6. 失败/取消留下无法确认归属或无法清理的公开半成品；
7. App卡死、任务永不终止、通知与页面终态冲突或错误成功。

不得用多段时间线、第二次有损编码、中间视频或关键帧强制吸附绕过问题。
