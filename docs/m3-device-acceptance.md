## 2. 每个成功输出都要检查

每次 App 显示成功后，都做以下检查：

- [ ] 页面显示了实际文件名和保存位置；
- [ ] 点击“打开”可以正常播放；
- [ ] 点击“分享”可以调出系统分享菜单；
- [ ] 从头、中间、结尾各听一次，没有爆音、变速、明显截断或长时间静音；
- [ ] 输出没有覆盖之前生成的同名文件；
- [ ] 任务完成后，常驻通知消失或变成正确的完成通知；
- [ ] 返回首页后可以开始下一次任务；
- [ ] 源视频仍然存在且可以正常播放；
- [ ] 没有其他非 VideoSlim 文件被删除或修改。

如果源文件、旧输出或其他文件被覆盖/删除，立即停止全部测试并报告，这是 blocker。

## 8. 取消测试

### 8.1 转换过程中取消

1. 用较长素材开始 AAC 128 kbps；
2. 进度到约 10%–30% 时，在 App 内点取消；
3. 取消按钮应进入“正在取消”，不要连续点击；
4. 等待明确的取消结果。

- [ ] 只出现一次取消结果；
- [ ] App 可以继续开始新任务；
- [ ] 没有可见的半成品；
- [ ] 源视频和旧输出都没有被删除。

再独立做一次，从系统通知点取消：

- [ ] 通知取消的是当前同一个任务；
- [ ] 最终没有重复通知或重复输出。

### 8.2 保存阶段取消

如果任务进度文字显示“正在保存/发布”，立即点一次取消；如果该阶段太快无法点到，写“未覆盖”，不要为了制造现场反复强杀 App。

- [ ] 取消后没有零字节或播不了的公开文件；
- [ ] 已经成功完成的旧输出没有被删除；
- [ ] 其他文件没有被删除；
- [ ] 页面没有先显示成功又改成取消，或先取消又生成错误成功。

## 11. 失败时怎么报告

遇到 FAIL 时，请提供：

1. 本清单中失败的章节和步骤；
2. 手机型号和系统版本；
3. 输入文件名、格式和大致时长；
4. 你点了什么、屏幕显示了什么；
5. 是否在后台、锁屏、取消或保存阶段发生；
6. 输出文件是否存在、大小是否为 0、能否播放；
7. 首页右上角小虫图标中的 F19 日志；
8. 如果方便，附屏幕录制或截图。

不要发送私人视频本身，除非你明确愿意。日志中的凭据、令牌或私人路径请替换为 `[REDACTED]`。

### 立即停止并优先报告的 blocker

- 真机上稳定复现 App 卡死或任务永不结束；
- App 报成功但没有可用输出；
- 源文件丢失、被覆盖或被修改；
- VideoSlim 之外的文件被删除；
- 已成功的旧输出被错误删除；
- 取消/失败后生成用户可见、无法清理的错误文件。
