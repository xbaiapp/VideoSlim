## 2. Pixel / GrapheneOS 优先阻塞门禁

先用 30–120 秒短片完成，全部通过后才运行约 98 分钟目标视频。

### 2.1 安装与基础权限

- [ ] APK SHA-256 与本文一致，并可覆盖安装旧私有验收版；
- [ ] App 可启动，不申请联网、读取全部媒体或所有文件访问权限；
- [ ] Android 13+ 通知权限拒绝时给出普通用户说明且不启动不可见任务；授予后任务正常；
- [ ] 系统相册 Photo Picker 和 SAF 文件选择均可导入；取消选择不报错；
- [ ] F19 可查看、复制和分享。

### 2.2 短片硬件路径

至少连续完成：

- [ ] 预设 HEVC 路径 3 次；
- [ ] 自定义 H.264 路径 2 次；
- [ ] 后台一次、锁屏一次；
- [ ] 如 HEVC 不可用，确认预设和自定义都在用户确认前不启动 H.264；
- [ ] 无可靠硬件 AVC Decoder 时快速阻止，不启动软件长任务。

每次检查 F19：

- [ ] 输入 AVC Decoder 候选包含当前尺寸、帧率和可获得的 profile/level；
- [ ] 输出 Encoder 候选包含目标尺寸、帧率、码率、Surface 与 VBR；
- [ ] 实际视频 Decoder/Encoder 与预检 allowlist 一致；
- [ ] F19 显示实际视频 Codec 为批准候选且 `hardware=true`、`software=false`；如平台硬件标志不明确，只能使用非已知软件名称的 vendor Codec；
- [ ] 普通页面和通知不显示 Media3、Codec、错误码或原始异常。

### 2.3 SAF 与 Provider

分别测试 Photo Picker、Media DocumentsProvider、ExternalStorage DocumentsProvider：

- [ ] F19 记录 raw read/persistable flags、持久授权尝试和 resolver 实际结果；
- [ ] 任务开始前 URI probe 可 open/read，大小与 metadata 基线一致；
- [ ] unknown size 不被单独判为 Provider 故障；不可 seek 的来源给出稳定、可理解结果；
- [ ] 移动/删除源文件后显示“文件已不可用”，不显示“文件损坏”；
- [ ] 授权消失时提示重新选择；
- [ ] Codec 失败后 URI 重开正常时保留 Decoder/Encoder 分类，不覆盖为源文件问题。

### 2.4 取消与发布 race

- [ ] 转码 10%–30% App 内取消：确认后立即显示“正在取消”，按钮锁定，只出现一个取消终态；
- [ ] 转码 80%–95% 通知取消：同一 taskId 被取消；
- [ ] 保存阶段快速重复点击/触发取消：请求幂等，等待保存或补偿清理结束；
- [ ] 取消后私有临时文件、`IS_PENDING` 行和公共半成品均不存在；
- [ ] 已成功输出及非 VideoSlim 文件不受影响；
- [ ] 取消调用失败时，页面保留期间收到的真实“保存中/取消中”阶段，不回退 ETA 或错误启用按钮。

### 2.5 约 98 分钟目标视频

短片门禁全部通过后，只执行一次完整目标视频：

- [ ] 输入约 97 分 54 秒、8.89 GB、720×1280、30 fps、SDR AVC High Profile Level 3.1；
- [ ] 先确认 F19 实际硬件 Decoder/Encoder；
- [ ] 任务覆盖前台→后台→锁屏并完整结束；
- [ ] 百分比单调，ETA 初期隐藏、停滞时重新估算、保存阶段不显示转码 ETA；
- [ ] 全程无 Released-state，无相同软件 Codec 原样自动重试；
- [ ] 输出可在系统相册播放，时长误差 <0.5 秒、音画同步，分辨率/码率/音轨符合设置；
- [ ] 完成后无残留通知、WakeLock、临时文件或待发布媒体行。

## 3. 通用 M2 参数矩阵

对典型 1080p/15 Mbps SDR 视频验证：

- [ ] 画质优先：HEVC、原分辨率、约 4 Mbps 基准、音频复制；
- [ ] 均衡：HEVC、原分辨率、约 2.5 Mbps 基准、音频复制；
- [ ] 极限压缩：长边 1280、约 1.2 Mbps 基准、AAC 96 kbps；
- [ ] 自定义分辨率：原始、1080p、720p、480p，较小来源不放大；
- [ ] 自定义编码：HEVC、H.264；
- [ ] 自定义视频码率：0.5、2.5、12 Mbps；
- [ ] 自定义音频：复制、AAC 192/128/96/64 kbps、移除；
- [ ] 参数变化后预估大小数量级正确；目标码率达到源视频码率 90% 时出现收益有限确认；
- [ ] HDR 输入明确提示转 SDR；支持设备色彩正常，不支持设备明确失败；
- [ ] 超过 6 小时或 50 GB 时出现超出已验证范围确认。

所有成功输出检查播放、时长、音画同步、画质、分辨率、码率和音频策略。

## 4. 后台、恢复和结果页

- [ ] 开始后出现常驻进度通知；App 与通知百分比基本一致且不回退；
- [ ] Home、切换 App、最近任务划走、锁屏后任务行为符合系统限制；
- [ ] 重新打开 Activity 后恢复同一 taskId、百分比、阶段和已用时间，不生成第二任务；
- [ ] 进程死亡不承诺断点续传；下次合法启动完成 recovery journal 对账；
- [ ] 发布中异常退出后无 `IS_PENDING=1` 行、无公共半成品；
- [ ] 连续 10 次异常中断后缓存和公共目录不持续增长，清理幂等且不误删；
- [ ] 成功页显示 `系统相册 > Movies > VideoSlim > 实际文件名`；
- [ ] 打开、分享、再次压缩正常；输出 metadata 回读失败时仍保留已保存位置和入口；
- [ ] Photo Picker 来源可在二次确认和系统授权后删除原视频；SAF 来源不显示该入口。

## 5. 扩展边界（不阻塞本次 98 分钟事故关闭）

- [ ] 接近 6 小时时长输入完整结束；
- [ ] 接近 50 GB 输入完整结束；
- [ ] 至少一个输出超过 4 GB，验证 muxer、MediaStore、文件系统、相册和播放器；
- [ ] 在 2–3 台不同 SoC/Android 设备重复后，才能扩大正式保证范围。

## 6. 通过与停止条件

本次 Pixel / GrapheneOS 事故关闭需同时满足：

1. 短片硬件路径、SAF/Provider、后台/锁屏和取消门禁通过；
2. F19 证明实际视频 Decoder/Encoder 均为批准的硬件候选；
3. 约 98 分钟目标视频完整成功且无 Released-state；
4. 失败、取消和发布补偿无持续存储增长或误删；
5. 所有普通用户提示可理解，技术细节只在 F19。

若硬件路径仍出现 Released-state，保留当前稳定代码并隔离测试新版 Media3，不直接替换正式路径；若失败时 URI 重开异常，优先处理 SAF/Provider，不继续调整 Codec。真机通过前不进入 M3。
