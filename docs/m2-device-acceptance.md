# VideoSlim M2 真机验收矩阵

> 状态：待 M2 APK 构建后由项目所有者在真机执行。本文只定义验收，不代表已通过。

## 1. 测试前记录

每台设备记录：

- 手机型号、SoC、Android 版本；
- 可用存储空间、总存储空间；
- 是否授予通知权限；
- 是否开启省电/超级省电模式；
- 测试 APK 版本、SHA-256；
- 每段输入的文件名、大小、时长、分辨率、帧率、视频/音频编码、码率、HDR/SDR。

如失败，复制 F19 全部日志并附失败发生阶段、屏幕状态和剩余存储空间。

## 2. 基础回归

- [ ] 系统相册 Photo Picker 选择视频正常；取消无错误。
- [ ] SAF 文件选择正常；取消无错误。
- [ ] 不申请读取媒体、全盘存储或联网权限。
- [ ] Android 8–9 仅在保存公共输出时申请旧版写权限。
- [ ] Android 13+ 首次任务按需申请通知权限；拒绝后给出可理解说明且不启动不可见任务。
- [ ] 文件信息与 MediaInfo 基本一致，竖拍显示方向正确。
- [ ] F19 日志查看、复制、分享正常。

## 3. F3 压缩参数

对同一段典型 1080p/15 Mbps SDR 视频验证：

- [ ] 画质优先：HEVC、保持原分辨率、约 4 Mbps 基准、音频复制。
- [ ] 均衡：HEVC、保持原分辨率、约 2.5 Mbps 基准、音频复制。
- [ ] 极限压缩：长边 1280、约 1.2 Mbps 基准、AAC 96 kbps。
- [ ] 自定义分辨率：保持原始、1080p、720p、480p。
- [ ] 自定义编码：HEVC、H.264。
- [ ] 自定义视频码率：0.5、2.5、12 Mbps 边界值。
- [ ] 自定义音频：复制、AAC 192/128/96/64 kbps、移除音轨。
- [ ] 参数变化后预估大小实时变化，数量级与实际输出一致。
- [ ] 目标视频码率达到源视频码率 90% 时出现“收益有限”确认。
- [ ] HEVC 硬件编码器不可用时明确降级 H.264，并将码率乘 1.5；无 H.264 时不启动。
- [ ] HDR 输入明确提示将转 SDR；支持设备输出色彩正常，不支持设备给出明确错误而非灰白/偏色成品。
- [ ] 时长超过 6 小时或源文件超过 50 GB 时出现“超出已验证范围”确认；取消确认不启动任务。

所有成功输出均检查：

- [ ] 能被系统相册和常用聊天应用播放；
- [ ] 时长误差小于 0.5 秒；
- [ ] 音画同步；
- [ ] 文字边缘无明显不可接受劣化；
- [ ] 已选择的 codec、分辨率、码率和音频策略与实际输出一致。

## 4. F6 前台服务、后台、熄屏与取消

### 4.1 普通长任务

- [ ] 开始后立刻出现常驻进度通知。
- [ ] 通知百分比与 App 内百分比基本一致且单调不回退。
- [ ] App 内显示已用时间和合理的预计剩余；初期可显示“计算中”，不得为负数或无限值。
- [ ] 按 Home、切换多个 App 后任务继续。
- [ ] 熄屏后任务继续，屏幕不被 App 强制点亮。
- [ ] 从通知返回 App 后恢复同一 taskId、百分比和已用时间，不产生第二个任务。
- [ ] 从最近任务列表划走 Activity 后，服务行为可预期；重新打开 App 可恢复仍在运行的同进程任务。
- [ ] 完成/失败/取消后通知、WakeLock、Transformer 和服务均结束。

### 4.2 取消

分别在预检、转码 10%～30%、转码 80%～95%、发布阶段执行：

- [ ] App 内取消需要确认；确认后按钮锁定，不能重复提交。
- [ ] 通知取消 action 可取消同一任务。
- [ ] 只收到一个 cancelled 终态。
- [ ] 私有临时文件被删除。
- [ ] 未完成 MediaStore 条目/公共半成品被删除。
- [ ] 已有成功输出和非本 App 文件不受影响。

### 4.3 边界输入（两个独立测试，不可互相替代）

- [ ] **时长边界**：使用时长接近 6 小时的输入；任务开始后切后台并熄屏，保持到完整结束。
- [ ] **体积边界**：使用源文件大小接近 50 GB 的输入；任务开始后切后台并熄屏，保持到完整结束。
- [ ] 至少一个测试形成超过 4 GB 的输出，验证 Media3 muxer、MediaStore、文件系统、系统相册和播放器。
- [ ] 记录实际墙钟耗时、平均速度、峰值温度/明显降频、完成时电量和剩余空间。

正式扩大 6 小时/50 GB 保证范围前，至少在 2–3 台不同 SoC、不同 Android 版本设备重复边界测试。

## 5. 异常退出和恢复清理

准备一个专门测试素材和空的 `Movies/VideoSlim` 测试基线：

- [ ] 转码中强制结束 App/进程；下次合法启动后任务不显示为仍运行。
- [ ] 发布中强制结束 App/进程；下次启动后无 `IS_PENDING=1` 条目、无公共半成品。
- [ ] `cache/transcode/` 中不存在未被活动任务引用的孤儿文件。
- [ ] 重复同类异常中断 10 次，私有缓存和公共目录占用不持续增长。
- [ ] 清理重复运行幂等。
- [ ] 已完成输出不被删除。
- [ ] 手工放入 `Movies/VideoSlim` 的非本 App 文件不被删除。
- [ ] 用户其他相册、下载目录和媒体完全不受影响。
- [ ] 清理失败时 App 仍可启动，F19 留有可读记录，下一次启动可重试。

说明：用户在系统设置中“强制停止”后，Android 不允许 App 自行后台重启。对账清理发生在用户下一次合法打开 App 或系统合法启动处理服务时。

## 6. F7 结果页

- [ ] 顶部醒目显示：`已保存到：系统相册 > Movies > VideoSlim > 实际文件名`。
- [ ] 用户无需自行搜索，10 秒内可通过主按钮打开/播放输出。
- [ ] 显示原大小、输出大小、节省字节和百分比；体积增加时如实显示。
- [ ] 系统分享可发送并播放输出。
- [ ] Photo Picker 相册来源显示“删除原视频”；二次确认和系统授权流程正确。
- [ ] SAF 文件来源不显示“删除原视频”。
- [ ] 删除原视频失败不影响已经生成的输出，并给出明确说明。
- [ ] 输出 metadata 回读失败时仍保留“已保存”、位置、文件名和打开/分享入口；绝不重新压缩生成重复输出。

## 7. 通过标准

M2 真机验收通过需同时满足：

1. F3、F6、F7 基础矩阵通过；
2. 时长边界和体积边界均通过，不以“熄屏 10 分钟”替代；
3. 正常失败、取消和 10 次异常中断不造成持续存储增长；
4. 未误删任何成功输出或非本 App 文件；
5. 至少完成一条 HDR 支持/不支持的诚实路径；
6. 所有阻塞性失败都有用户可读文案和 F19 原始链路。

验收通过前不进入 M3。
